# HOW TO WRITE REQUIREMENTS FOR OPTIMAL RESULTS

# Write requirements in pipeline_data/requirements.txt following these rules:
#
# 1. CLEAR DESCRIPTION
#    Start with one sentence describing what the application does.
#
# 2. LIST FEATURES
#    Specify each feature in detail.
#    Format:
#    Features:
#    - [feature 1 with details]
#    - [feature 2 with details]
#    - [feature N with details]
#
# 3. SPECIFY DEPLOYMENT PLATFORM (CRITICAL!)
#    Choose where to deploy. FREE options:
#    - github-pages: static sites only (no backend, no database)
#    - vercel: React/Next.js + serverless functions
#    - railway: fullstack with database (recommended, $5 free credit/month)
#    - render: fullstack with database (free but with cold starts)
#    - azure-free: Azure Static Web Apps (if you have Azure credits)
#
#    Format: "Deploy to: railway"
#
# 4. TECHNICAL REQUIREMENTS
#    Specify preferred technologies:
#    - Frontend: React, Vue, Angular
#    - Backend: .NET, Node.js, Python, or "none"
#    - Database: PostgreSQL, MySQL, MongoDB, or "none"
#
#    Format:
#    Technical:
#    - Frontend: [framework]
#    - Backend: [framework]
#    - Database: [database or none]
#    - Deploy to: [platform]
#
# 5. USER STORIES (optional but recommended)
#    Describe what users want to do:
#    Format:
#    User stories:
#    - As a [user type], I want to [action] so that [benefit]
#
