# MCP Servers - Integration Layer

This directory contains **MCP (Model Context Protocol) servers** that provide tool integrations for GitHub, Azure DevOps, Railway, and Docker deployments.

---

## 🏗️ Architecture

```
Backend FastAPI
    ↓ (injects encrypted tokens as env vars)
MCPClientManager
    ↓ (stdio connections)
┌──────────────┬─────────────────────┬──────────────┐
│ GitHub       │ Azure DevOps        │ Deploy       │
│ Server       │ Server              │ Server       │
│ Port: 8001   │ Port: 8002          │ Port: 8003   │
└──────────────┴─────────────────────┴──────────────┘
```

### Key Concepts

1. **Servers run as separate processes** - Each MCP server is an independent Python process
2. **Stdio communication** - Servers communicate via stdin/stdout (MCP protocol)
3. **Token injection** - Backend reads encrypted tokens from DB, decrypts them, injects as env vars before starting servers
4. **User never interrupted** - All authentication happens before agent execution starts

## 🚀 Starting Servers

### GitHub Server

```bash
# Set environment variable (injected by backend at runtime)
export GITHUB_TOKEN="ghp_your_token_here"

# Start server
python -m mcp_servers.github_server
```

**Available Tools**:
- `create_repository(name, private, description)` - Create a new repo
- `push_files(repo, branch, files, message)` - Push multiple files in one commit
- `create_pull_request(repo, title, head, base, body)` - Create PR
- `read_file(repo, path, ref)` - Read file content from repo

---

### Azure DevOps Server

```bash
# Set environment variables
export AZDO_TOKEN="your_pat_token"
export AZDO_ORG="https://dev.azure.com/yourorg"

# Start server
python -m mcp_servers.azuredevops_server
```

**Available Tools**:
- `create_work_item(project, type, title, description, tags)` - Create User Story, Task, Bug, etc.
- `create_sprint(project, name, start_date, end_date)` - Create sprint/iteration
- `create_pipeline(project, name, yaml_path, repo_url)` - Create CI/CD pipeline

---

### Deploy Server

```bash
# Set environment variable (Railway)
export RAILWAY_TOKEN="your_railway_token"

# Start server
python -m mcp_servers.deploy_server
```

**Available Tools**:
- `deploy_railway(project_name, env_vars, dockerfile_path)` - Deploy to Railway
- `deploy_docker_compose(compose_content, target_host)` - Deploy with Docker Compose

## 🔌 Using MCP Client Manager

The `MCPClientManager` provides a unified interface to interact with all servers:

```python
from mcp_servers.client import MCPClientManager

# Create manager
manager = MCPClientManager()

# Connect to servers
await manager.connect("github")
await manager.connect("azuredevops")

# Get available tools
tools = await manager.get_tools(["github", "azuredevops"])
# Returns list of all tools from both servers

# Execute a tool
result = await manager.call_tool(
    server="github",
    tool_name="create_repository",
    args={
        "name": "my-new-app",
        "private": False,
        "description": "Generated by Agentic Orchestra"
    }
)

# Cleanup
await manager.disconnect_all()
```

## 🔐 How Authentication Works

### 1. User Connects Account (One-Time, via UI)

User goes to UI Settings → Integrations → Connect GitHub:
- For GitHub: OAuth flow (already implemented in `orchestrator-ui/backend/api/auth.py`)
- For others: User enters PAT token manually

Token is encrypted and stored in DB:
```python
# orchestrator-ui/backend/encryption_service.py
encrypted_token = encrypt_token(user_token)

# Stored in DeployProviderAuth model
db.add(DeployProviderAuth(
    user_id=user.id,
    provider="github",
    encrypted_token=encrypted_token
))
```

### 2. Backend Injects Tokens at Runtime

When agent starts generation:
```python
# orchestrator-ui/backend/orchestrator.py (pseudocode)

# Read encrypted tokens from DB
github_auth = db.query(DeployProviderAuth).filter_by(
    user_id=user.id,
    provider="github"
).first()

# Decrypt
github_token = decrypt_token(github_auth.encrypted_token)

# Inject as environment variable
os.environ["GITHUB_TOKEN"] = github_token

# Start MCP servers (they inherit env vars)
await mcp_client.connect("github")
```

### 3. Server Reads Token

```python
# mcp_servers/github_server.py
from .base_server import inject_token

def get_github_client():
    token = inject_token("GITHUB_TOKEN")  # Reads from environment
    return Github(token)
```

**If token missing**:
```python
raise MCPAuthError(
    "Missing authentication: GITHUB_TOKEN environment variable not set. "
    "User must connect their account via the UI first."
)
```

## ➕ Adding a New MCP Server

### Step 1: Create Server File

Create `mcp_servers/your_service_server.py`:

```python
from mcp.server import Server
from mcp.types import Tool, TextContent
from .base_server import inject_token, log_tool_call, format_success_response

app = Server("your-service-server")

@app.list_tools()
async def list_tools() -> list[Tool]:
    return [
        Tool(
            name="your_tool",
            description="What your tool does",
            inputSchema={
                "type": "object",
                "properties": {
                    "arg1": {"type": "string", "description": "..."}
                },
                "required": ["arg1"]
            }
        )
    ]

@app.call_tool()
@log_tool_call
async def call_tool(name: str, arguments: dict) -> list[TextContent]:
    if name == "your_tool":
        result = await your_tool_implementation(**arguments)
        return [TextContent(type="text", text=str(format_success_response(result)))]

async def your_tool_implementation(arg1: str) -> dict:
    token = inject_token("YOUR_SERVICE_TOKEN")
    # ... implementation ...
    return {"status": "success", "data": ...}

if __name__ == "__main__":
    import asyncio
    import mcp

    async def main():
        async with mcp.server.stdio.stdio_server() as (read_stream, write_stream):
            await app.run(read_stream, write_stream, app.create_initialization_options())

    asyncio.run(main())
```

### Step 2: Register in Client Manager

Edit `mcp_servers/client.py`:

```python
class MCPClientManager:
    SERVERS = {
        # ... existing servers ...
        "your_service": {
            "command": "python",
            "args": ["-m", "mcp_servers.your_service_server"],
            "description": "Your service integration"
        }
    }
```

### Step 3: Add DB Model (if new provider)

Edit `orchestrator-ui/backend/models.py`:

```python
# Add to DeployProvider enum
class DeployProvider(str):
    GITHUB = "github"
    RAILWAY = "railway"
    YOUR_SERVICE = "your_service"  # Add this
```

### Step 4: Add UI Integration

Add UI form in `orchestrator-ui/frontend/src/screens/SettingsScreen.tsx` for users to connect account.

### Step 5: Update Backend Token Injection

Edit `orchestrator-ui/backend/orchestrator.py` to inject `YOUR_SERVICE_TOKEN` before starting servers.

## 🧪 Testing Servers

### Unit Test a Server

```python
# test_github_server.py
import os
os.environ["GITHUB_TOKEN"] = "test_token"

from mcp_servers.github_server import create_repository

result = await create_repository(
    name="test-repo",
    private=False,
    description="Test repository"
)

assert result["name"] == "test-repo"
```

### Integration Test with Client Manager

```python
# test_mcp_integration.py
from mcp_servers.client import MCPClientManager

manager = MCPClientManager()
await manager.connect("github")

tools = await manager.get_tools(["github"])
assert any(t["name"] == "create_repository" for t in tools)

result = await manager.call_tool(
    "github",
    "create_repository",
    {"name": "integration-test-repo"}
)

assert result["success"] is True
```

## 🛡️ Error Handling

All servers use standardized error responses:

### Authentication Error
```json
{
  "error": "authentication_required",
  "message": "Missing authentication: GITHUB_TOKEN environment variable not set. User must connect their account via the UI first.",
  "code": "AUTH_MISSING"
}
```

### Tool Error
```json
{
  "error": "ValueError",
  "message": "Repository name cannot be empty",
  "code": "TOOL_ERROR"
}
```

### Success Response
```json
{
  "success": true,
  "data": {
    "id": 123,
    "name": "my-repo",
    "url": "https://github.com/user/my-repo"
  }
}
```
